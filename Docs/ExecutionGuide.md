## 2. Important: Identity Column Setup

Oracle tables use `GENERATED ALWAYS AS IDENTITY` by default. To insert seed data with explicit IDs, you must first alter each table:

```sql
-- Run BEFORE seeding (for each table with explicit ID inserts):
ALTER TABLE Department MODIFY (department_id GENERATED BY DEFAULT ON NULL AS IDENTITY);
-- Repeat for: Doctor, Staff, Patient, Insurance, DoctorSchedule, DoctorVacation,
--             Room, Bed, Appointment, Admission, Prescription, Billing, Payment

-- Run AFTER seeding to restore auto-generation:
ALTER TABLE Department MODIFY (department_id GENERATED ALWAYS AS IDENTITY START WITH LIMIT VALUE);
-- Repeat for all tables above
```

Each DML seed file includes these instructions in comments at the top.

## 5. Troubleshooting

### "ORA-01031: insufficient privileges"
- **Cause:** Not connected as DBA for setup steps.
- **Fix:** Connect as `SYS AS SYSDBA` or `SYSTEM` for Steps 1, 2, and 6.

### "ORA-02291: integrity constraint violated - parent key not found"
- **Cause:** DML scripts executed out of order.
- **Fix:** Follow the DML execution order exactly (7a through 7i). Parent data must exist before child inserts.

### "ORA-32795: cannot insert into a generated always identity column"
- **Cause:** Identity columns not altered before seeding.
- **Fix:** Run `ALTER TABLE <table> MODIFY (<id_col> GENERATED BY DEFAULT ON NULL AS IDENTITY);` before inserting with explicit IDs.

### "ORA-04098: trigger is invalid and failed re-validation"
- **Cause:** Trigger references a table or column that doesn't exist yet.
- **Fix:** Ensure all tables are created (Phase 1) and seeded (Phase 2) before creating triggers (Phase 4).

### "PLS-00201: identifier must be declared" when running procedures
- **Cause:** Functions not created yet.
- **Fix:** Run fn_IsDoctorOnVacation.sql and fn_GetDoctorDailyCount.sql (Steps 8a, 8b) before any stored procedures.

### DBMS_OUTPUT not showing messages
- **Cause:** Server output not enabled.
- **Fix:** Run `SET SERVEROUTPUT ON;` at the start of your session (SQL*Plus/SQL Developer).

### "ORA-00001: unique constraint violated" during seeding
- **Cause:** Seed script was run more than once.
- **Fix:** Truncate or delete existing data before re-running, or drop and recreate the table.
