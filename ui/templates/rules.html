Business Rules

BR1 / TV1 No Duplicate Booking

Enforced in two layers: stored procedure and database constraint.
What
A patient cannot hold two Scheduled appointments on the same DoctorSchedule slot.
Procedure
sp_BookAppointment — counts existing Scheduled rows for (patient_id, schedule_id) and raises ORA-20004 on match.
Constraint
UNIQUE(patient_id, schedule_id) on Appointment — catches any race condition that bypasses the procedure check.
Test
Tests/TC01_DuplicateBooking.sql

BR2 24-Hour Cancellation Rule

Enforced in sp_CancelAppointment.
What
An appointment may be cancelled only if its slot is at least 24 hours in the future.
Mechanism
The procedure assembles the slot datetime from schedule_date + slot_time and computes (v_appt_datetime - SYSDATE) * 24. If the result is < 24, it raises ORA-20002 and rolls back.
On violation
Appointment status remains Scheduled; no row is updated.
Test
Tests/TC02_CancelBefore24hr.sql

BR3 Rescheduling Preserves History

Enforced by the procedure and reinforced by a trigger.
What
Every reschedule must leave an audit trail in AppointmentHistory that captures the previous slot and a change reason.
Procedure
sp_RescheduleAppointment — after the UPDATE, inserts a row with previous_schedule_id, previous_status, new_status, and the caller-provided change_reason.
Trigger
trg_AppointmentHistory fires on status changes only (e.g. Scheduled → Cancelled / Completed). Pure schedule-id changes are intentionally skipped to avoid double-logging when the procedure runs.
Test
Tests/TC05_RescheduleHistory.sql

BR5 Vacation Blocking

Enforced by the procedure via a helper function.
What
A doctor on an approved vacation cannot receive new bookings for any date in that vacation window.
Mechanism
fn_IsDoctorOnVacation(employee_id, date) returns 1 when an approved vacation row covers the date. sp_BookAppointment and sp_RescheduleAppointment call it before inserting and raise ORA-20003 on match.
Note
Only vacations with status = 'Approved' block bookings. Pending / Rejected rows are ignored.
Test
Tests/TC04_VacationBlocking.sql

BR6 Doctor Schedule Overlap Prevention

Enforced at the database level.
What
A single doctor cannot have two schedule slots on the same date at the same start time.
Constraint
UNIQUE(employee_id, schedule_date, slot_time) on DoctorSchedule — any duplicate insert raises ORA-00001.
Test
Tests/TC08_ScheduleOverlap.sql

Transaction Validations

TV1 Duplicate Booking Prevention (Transactional)

Same layers as BR1, viewed under concurrency.
Scenario
Two sessions call sp_BookAppointment for the same patient + slot near-simultaneously.
Defense 1
The procedure-level SELECT COUNT check inside the implicit transaction.
Defense 2
The UNIQUE(patient_id, schedule_id) constraint catches any race that slips past Defense 1.
Outcome
One insert succeeds; the other raises ORA-20004 (procedure) or ORA-00001 (constraint) and rolls back.

TV2 Occupied Bed Assignment Must Fail

Enforced by trg_PreventOccupiedBed.
Scenario
INSERT INTO Admission targets a bed_id that already has an Active admission.
Mechanism
BEFORE INSERT row trigger counts active admissions on the target bed. If any exist, it raises ORA-20002 'Bed is currently occupied...' and aborts the insert.
Note
Oracle does not support INSTEAD OF triggers on tables, so this is a BEFORE INSERT guard rather than a row-rewrite.
Test
Tests/TC06_OccupiedBedFail.sql

TV3 Insurance Discount on Billing

Enforced by two mirror triggers after the Billing-table fold-in.
Scenario
An Appointment or Admission has bill_amount set or changed for a patient with active insurance.
Mechanism
trg_AppointmentInsDiscount and trg_AdmissionInsDiscount (BEFORE UPDATE OF bill_amount) look up the patient's insurance. If is_active = 1 and valid_until >= TRUNC(SYSDATE), they compute bill_amount × coverage_percentage / 100, cap at max_coverage_amount, and write the result back into insurance_coverage_amt.
Inactive / missing policy
insurance_coverage_amt = 0.
Demo
Observe via the Complete + bill flow — insurance_coverage_amt is populated automatically by the trigger when sp_CompleteAppointment sets bill_amount.
